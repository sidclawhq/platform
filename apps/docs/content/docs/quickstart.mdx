# Quick Start

## The fast path (recommended)

Create a fully working governed agent with one command:

```bash
npx create-sidclaw-app my-governed-agent
```

The CLI will:
1. Ask you to pick a **framework** (LangChain, Vercel AI, OpenAI Agents, MCP Proxy, plain TS/Python)
2. Open the **signup page** if you don't have an account yet — create one, then go to **Settings > API Keys > Create Key**
3. Ask you to **paste your API key**
4. Create a **registered agent** and **3 demo policies** in your account
5. Scaffold a project with the SDK pre-configured

Then run it:

```bash
cd my-governed-agent
npm start   # or: python main.py (for Python templates)
```

### What just happened

Your project includes 3 demo tools, each hitting a different policy:

| Tool | Policy | What you see |
|------|--------|-------------|
| `search_docs` | **allow** | Executes immediately. The trace appears in the [audit log](https://app.sidclaw.com/dashboard/audit). |
| `send_email` | **approval_required** | Paused. Open the [approval queue](https://app.sidclaw.com/dashboard/approvals) to see the request, review the context, and approve or deny. |
| `export_data` | **deny** | Blocked before execution. The SDK throws `ActionDeniedError`. |

This is the core loop: **Identity > Policy > Approval > Trace**. Every agent action is evaluated, and every decision is recorded.

> **Running inside NemoClaw?** If you're deploying your agent in an NVIDIA NemoClaw sandbox, see the [NemoClaw integration guide](/docs/integrations/nemoclaw) for sandbox-specific setup.

## Add to an existing project (TypeScript)

Use this path if you already have an agent and want to add SidClaw governance to it.

### 1. Install the SDK

```bash
npm install @sidclaw/sdk
```

### 2. Get an API key

Sign up at [app.sidclaw.com](https://app.sidclaw.com/signup), then go to **Settings > API Keys > Create Key**.

<details>
<summary>Running locally (self-hosted)?</summary>

The key is printed when you seed the database:

```bash
cd apps/api && npx prisma db seed
# → API key: sk_dev_...
```

The seed key is also saved to `deployment/.env.development`.

</details>

### 3. Create an agent

Go to [app.sidclaw.com/dashboard/agents](https://app.sidclaw.com/dashboard/agents) and click **Create Agent**. Fill in a name, owner, and environment. Copy the agent ID (e.g., `ag_abc123`).

### 4. Create a policy

Go to [app.sidclaw.com/dashboard/policies](https://app.sidclaw.com/dashboard/policies) and click **Create Policy**. Create at least one policy that matches the operations your agent will perform. Without a policy, all actions are denied by default (secure-by-default).

For example, to require approval for email sending:
- **Operation:** `send_email`
- **Effect:** `approval_required`
- **Agent:** select your agent from step 3

### 5. Wrap your agent's tools

```typescript
import { AgentIdentityClient, withGovernance } from '@sidclaw/sdk';

const client = new AgentIdentityClient({
  apiKey: process.env.SIDCLAW_API_KEY!,
  apiUrl: 'https://api.sidclaw.com',  // or http://localhost:4000
  agentId: process.env.SIDCLAW_AGENT_ID!,
});

const sendEmail = withGovernance(client, {
  operation: 'send_email',
  target_integration: 'email_service',
  resource_scope: 'customer_emails',
  data_classification: 'confidential',
}, async (to: string, subject: string, body: string) => {
  await emailService.send({ to, subject, body });
});

// Now governed — call it normally:
await sendEmail('customer@example.com', 'Follow-up', 'Hello...');
```

When your agent calls `sendEmail`, the SDK:

1. Sends the action details to `POST /api/v1/evaluate`
2. The Policy Engine matches the action against your rules
3. Based on the policy decision:
   - **`allow`** — Executes the function immediately and records the outcome
   - **`approval_required`** — Polls until a human reviewer approves or denies, then executes or throws
   - **`deny`** — Throws `ActionDeniedError` without executing

### 6. Handle errors

```typescript
import { ActionDeniedError, ApprovalExpiredError } from '@sidclaw/sdk';

try {
  await sendEmail('customer@example.com', 'Follow-up', 'Hello...');
} catch (error) {
  if (error instanceof ActionDeniedError) {
    console.log('Blocked by policy:', error.reason);
    console.log('Trace ID:', error.traceId);
  }
  if (error instanceof ApprovalExpiredError) {
    console.log('No reviewer responded in time');
  }
}
```

### 7. See it in action

Open the [dashboard](https://app.sidclaw.com) to see approval requests and audit traces in real-time.

## Add to an existing project (Python)

### 1. Install

```bash
pip install sidclaw
```

### 2. Set up your account

Follow the same steps as TypeScript above: [get an API key](#2-get-an-api-key), [create an agent](#3-create-an-agent), and [create a policy](#4-create-a-policy).

### 3. Govern your tools

```python
import os
from sidclaw import SidClaw
from sidclaw.middleware.generic import with_governance, GovernanceConfig

client = SidClaw(
    api_key=os.environ["SIDCLAW_API_KEY"],
    agent_id=os.environ["SIDCLAW_AGENT_ID"],
)

@with_governance(client, GovernanceConfig(
    operation="send_email",
    target_integration="email_service",
    data_classification="confidential",
))
def send_email(to, subject, body):
    email_service.send(to=to, subject=subject, body=body)

send_email("customer@example.com", "Follow-up", "Hello...")
```

### 4. Handle errors

```python
from sidclaw import ActionDeniedError, ApprovalExpiredError

try:
    send_email("customer@example.com", "Follow-up", "Hello...")
except ActionDeniedError as e:
    print(f"Blocked by policy: {e.reason}")
    print(f"Trace ID: {e.trace_id}")
except ApprovalExpiredError:
    print("No reviewer responded in time")
```

### 5. See it in action

Open the [dashboard](https://app.sidclaw.com). The experience is the same as with the TypeScript SDK.
