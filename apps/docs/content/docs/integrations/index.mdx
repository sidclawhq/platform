# Integrations

SidClaw provides middleware integrations for major AI agent frameworks and notification channels for approval workflows. Each integration follows the same pattern: wrap your existing tools with a one-liner, and SidClaw handles policy evaluation, approval routing, and audit logging.

## AI Agent Frameworks

Govern tool calls across the most popular agent frameworks with zero changes to your existing code.

| Integration | TypeScript | Python | Pattern |
|---|---|---|---|

## Protocol & Platform Integrations

| Integration | Description |
|---|---|
| [MCP](/docs/integrations/mcp) | Governance proxy for Model Context Protocol servers. Intercepts tool calls, enforces policies, routes approvals. Available as CLI binary (`sidclaw-mcp-proxy`) or programmatic API. |
| [MCP Tools Server](/docs/integrations/mcp-tools) | `@sidclaw/mcp-tools` -- an MCP server that exposes SidClaw governance primitives (`sidclaw_evaluate`, `sidclaw_record`, `sidclaw_approve`) as callable tools. |
| [NemoClaw](/docs/integrations/nemoclaw) | Govern NVIDIA NemoClaw sandbox tools. Also available as SDK middleware (see Agent Frameworks above) and can generate MCP-compatible proxy configurations. |
| [OpenClaw](/docs/integrations/openclaw) | Published as `sidclaw-governance` skill on ClawHub. Governs any OpenClaw skill with policy enforcement and audit trails. |
| [OpenClaw Lifecycle Plugin](/docs/integrations/openclaw-plugin) | `@sidclaw/openclaw-plugin` -- drop-in plugin that hooks OpenClaw agent lifecycle events (`before_tool_call`, `llm_output`, etc.) without changing agent code. |
| [GitHub Copilot](/docs/integrations/github-copilot) | Governance for GitHub Copilot Studio and Copilot agents via HTTP transport. |
| [Copilot Studio](/docs/integrations/copilot-studio) | Governance for Microsoft Copilot Studio skills via OpenAPI action integration. |

## Notification Channels

SidClaw delivers approval requests to your team's preferred communication channels. Reviewers can approve or deny directly from chat.

| Channel | Description |
|---|---|
| [Slack](/docs/integrations/slack) | Block Kit messages with interactive Approve/Deny buttons. Messages update in-place after decision. |
| [Microsoft Teams](/docs/integrations/teams) | Adaptive Card notifications with Approve/Deny buttons (Bot Framework mode) or dashboard links (webhook mode). |
| [Telegram](/docs/integrations/telegram) | HTML messages with inline keyboard buttons. Callback updates remove buttons and add reply. |
| [Resend](/docs/integrations/resend) | Email notifications for approval requests via Resend transactional email. |

## Quick Start

Every framework integration follows the same three steps:

1. **Install the SDK** -- `npm install @sidclaw/sdk` or `pip install sidclaw`
2. **Wrap your tools** -- one function call per framework
3. **Configure policies** -- in the SidClaw dashboard

```typescript
import { governTools } from "@sidclaw/sdk/langchain";

// Before: const tools = [searchTool, emailTool];
// After:
const tools = governTools(client, agentId, [searchTool, emailTool]);
```

See individual integration pages for framework-specific examples.
