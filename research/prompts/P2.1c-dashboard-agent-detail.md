### Component Specs

#### AgentDetailHeader
- Agent name: `text-xl font-medium text-primary`
- Description: `text-sm text-secondary mt-1`
- Lifecycle badge: reuse `AgentLifecycleBadge` from P2.1b
- Action buttons (right-aligned):
  - If active: show **Suspend** (amber outline) and **Revoke** (red outline)
  - If suspended: show **Reactivate** (green outline) and **Revoke** (red outline)
  - If revoked: no action buttons (terminal state), show "This agent has been permanently revoked" in `text-accent-red text-sm`
- Buttons: `border px-4 py-1.5 rounded text-sm font-medium`

#### AgentOverviewSection
- Card: `bg-surface-1 border-default rounded-lg p-5`
- Key-value pairs using a consistent label/value layout:
  - Label: `text-xs text-muted uppercase tracking-wider`
  - Value: `text-sm text-primary`
- Fields: Owner, Role, Team, Environment (badge), Created, Next Review Date

#### AgentAuthoritySection
- Same card style as Overview
- Fields: Authority Model, Identity Mode, Delegation Model, Autonomy Tier
- Autonomy tier badge: low = green, medium = amber, high = red (risk indicator)

#### AgentIntegrationsTable
- Card with inner table
- Columns: Integration Name, Resource Scope, Data Classification (badge), Allowed Operations
- Data classification badges: same as approval queue (public=gray, internal=blue, confidential=amber, restricted=red)
- Operations: comma-separated list in `font-mono text-xs`
- If no integrations: "No authorized integrations configured"

#### AgentPolicySummary
- Card showing 3 stat numbers with effect badges:
  - Allow: green number
  - Approval Required: amber number
  - Deny: red number
- Link: "View all policies →" navigates to `/dashboard/policies?agent_id=[id]`

#### AgentRecentActivity
- Card with two sub-sections:
  - Stats: traces last 7 days (number), pending approvals (number, amber if > 0)
  - Recent traces list (last 5): operation → integration, outcome badge, relative time
    - Click navigates to `/dashboard/audit?trace=[trace_id]` (or highlights in trace viewer)
  - Recent approvals (last 3): operation, status badge, relative time
    - Click navigates to `/dashboard/approvals` (with approval ID if deep linking exists)
- Link: "View all traces →" navigates to `/dashboard/audit?agent_id=[id]`

#### AgentLifecycleControls & LifecycleConfirmDialog
- Buttons trigger a confirmation dialog before calling the API
- Dialog content varies by action:
  - **Suspend**: "Suspend [Agent Name]? All pending evaluations will be denied. The agent can be reactivated later."
  - **Revoke**: "Revoke [Agent Name]? This is permanent — the agent cannot be reactivated. All pending evaluations will be denied."
  - **Reactivate**: "Reactivate [Agent Name]? The agent will resume normal policy evaluation."
- Dialog: `bg-surface-1 border-default rounded-lg p-6`, backdrop `bg-black/50`
- Confirm button matches the action color (suspend=amber, revoke=red, reactivate=green)
- Cancel button: `bg-surface-2 text-secondary`
- On confirm: call API, close dialog, show toast ("Agent suspended" / "Agent revoked" / "Agent reactivated"), refresh page data
- On error: show toast with error message
