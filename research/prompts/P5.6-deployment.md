## What To Do

### 1. API Dockerfile (`apps/api/Dockerfile`)

Multi-stage build for minimal image size:

```dockerfile
# Stage 1: Install dependencies
FROM node:20-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
COPY packages/shared/package.json ./packages/shared/
COPY apps/api/package.json ./apps/api/
RUN npm ci --workspace=apps/api --workspace=packages/shared

# Stage 2: Build
FROM node:20-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY --from=deps /app/packages/shared/node_modules ./packages/shared/node_modules
COPY --from=deps /app/apps/api/node_modules ./apps/api/node_modules
COPY packages/shared/ ./packages/shared/
COPY apps/api/ ./apps/api/
COPY tsconfig.base.json ./
# Generate Prisma client
RUN cd apps/api && npx prisma generate
# Build shared
RUN cd packages/shared && npm run build
# Build API
RUN cd apps/api && npm run build

# Stage 3: Production
FROM node:20-alpine AS production
WORKDIR /app
RUN addgroup --system --gid 1001 nodejs && \
    adduser --system --uid 1001 api
COPY --from=build /app/apps/api/dist ./dist
COPY --from=build /app/apps/api/prisma ./prisma
COPY --from=build /app/apps/api/node_modules ./node_modules
COPY --from=build /app/apps/api/package.json ./package.json
# Copy Prisma client
COPY --from=build /app/node_modules/.prisma ./node_modules/.prisma
COPY --from=build /app/node_modules/@prisma ./node_modules/@prisma

USER api

# Health checks
HEALTHCHECK --interval=10s --timeout=3s --start-period=30s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:4000/health || exit 1

EXPOSE 4000

# Run migrations then start server
CMD ["sh", "-c", "npx prisma migrate deploy && node dist/server.js"]
```

**Note:** The exact COPY paths depend on the monorepo structure and how TypeScript builds. Adjust based on actual build output. The key principles are:
- Multi-stage to minimize image size
- Prisma client generated in build stage
- Migrations run on container start (before server)
- Non-root user
- Health check built-in

### 3. Docs Dockerfile (`apps/docs/Dockerfile`)

```dockerfile
FROM node:20-alpine AS build
WORKDIR /app
COPY apps/docs/ ./
RUN npm ci && npm run build

FROM nginx:alpine AS production
COPY --from=build /app/out /usr/share/nginx/html
COPY apps/docs/nginx.conf /etc/nginx/conf.d/default.conf

HEALTHCHECK --interval=10s --timeout=3s --retries=3 \
  CMD wget --no-verbose --tries=1 --spider http://localhost:80 || exit 1

EXPOSE 80
```

Create `apps/docs/nginx.conf`:

```nginx
server {
    listen 80;
    root /usr/share/nginx/html;
    index index.html;

    location / {
        try_files $uri $uri.html $uri/index.html =404;
    }

    location /_next/static {
        expires 365d;
        add_header Cache-Control "public, immutable";
    }

    error_page 404 /404.html;
}
```

**Note:** Requires `output: 'export'` in docs `next.config.ts` if using Next.js static export. If Fumadocs doesn't support static export, use the standalone approach instead.

### 5. Production Docker Compose (`docker-compose.production.yml`)

```yaml
version: '3.8'

services:
  db:
    image: postgres:16-alpine
    environment:
      POSTGRES_DB: ${DB_NAME:-sidclaw}
      POSTGRES_USER: ${DB_USER:-sidclaw}
      POSTGRES_PASSWORD: ${DB_PASSWORD:?DB_PASSWORD is required}
    volumes:
      - pgdata:/var/lib/postgresql/data

    build:

### 7. Liveness Probe Endpoint

app.get('/health/live', async (request, reply) => {

});
```

### 8. Startup Validation

Update `apps/api/src/server.ts` to validate critical configuration on startup:

```typescript
// In production, fail fast if critical config is missing
if (config.environment === 'production') {
  const required = ['databaseUrl', 'sessionSecret'];
  const missing = required.filter(key => !config[key]);
  if (missing.length > 0) {
    console.error(`FATAL: Missing required configuration: ${missing.join(', ')}`);
    process.exit(1);
  }

  if (config.sessionSecret && config.sessionSecret.length < 32) {
    console.error('FATAL: SESSION_SECRET must be at least 32 characters');
    process.exit(1);
  }
}
```
