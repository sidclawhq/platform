### 3. Quickstart Page (Most Important)

```markdown
---
title: Quick Start
description: Get governance running on your AI agent in 2 minutes.
---

# Quick Start

Get governance running on your AI agent in 2 minutes.

## 1. Install the SDK

\`\`\`bash
npm install @sidclaw/sdk
\`\`\`

## 2. Get an API Key

[Sign up for free](https://sidclaw.com/signup) → go to Settings → API Keys → Create Key.

Or if running locally: the API key is printed during `npx prisma db seed`.

## 3. Initialize the Client

\`\`\`typescript
import { AgentIdentityClient } from '@sidclaw/sdk';

const client = new AgentIdentityClient({
  apiKey: process.env.SIDCLAW_API_KEY!,
  apiUrl: 'https://api.sidclaw.com',  // or http://localhost:4000
  agentId: 'your-agent-id',
});
\`\`\`

## 4. Wrap Your Agent's Tools

\`\`\`typescript
import { withGovernance } from '@sidclaw/sdk';

const sendEmail = withGovernance(client, {
  operation: 'send_email',
  target_integration: 'email_service',
  resource_scope: 'customer_emails',
  data_classification: 'confidential',
}, async (to: string, subject: string, body: string) => {
  // your actual email sending logic
  await emailService.send({ to, subject, body });
});

// Now governed:
await sendEmail('customer@example.com', 'Follow-up', 'Hello...');
// → Policy says "allow"? Executes immediately.
// → Policy says "approval_required"? Waits for human approval.
// → Policy says "deny"? Throws ActionDeniedError.
\`\`\`

## 5. See It in Action

1. Run your agent
2. Open the [dashboard](https://app.sidclaw.com/dashboard/approvals) (or http://localhost:3000)
3. If the action required approval: you'll see an approval card with full context
4. Approve it — the agent continues
5. Check the [trace viewer](https://app.sidclaw.com/dashboard/audit) to see the complete audit trail
```

### 4. Compliance Mapping Pages

These are critical for enterprise GTM. Each page maps regulatory requirements to specific product capabilities.

```markdown

title: FINRA 2026 Compliance
description: How SidClaw addresses FINRA 2026 AI agent governance requirements
---

# FINRA 2026 Compliance Mapping

FINRA's 2026 Annual Regulatory Oversight Report explicitly requires governance controls
for AI agents in financial services. Here's how SidClaw maps to each requirement.

## Pre-Approval of AI Use Cases

**FINRA requires:** Documented sign-offs and defined supervisory owners for AI use cases.

**SidClaw provides:**
- **Agent Registry** — every agent is registered with an owner, authority model, and scoped permissions
- **Policy Rules** — every action is evaluated against explicit policies with documented rationale
- **Audit Trail** — every evaluation creates a trace with the policy that was applied

## Human-in-the-Loop Validation

**FINRA requires:** "For any AI output that influences a decision or touches a client,
there must be a documented human checkpoint."

**SidClaw provides:**
- **Approval Primitive** — policies can require human approval before execution
- **Context-Rich Approval Cards** — reviewers see what the agent wants to do, why it was flagged,
  the agent's reasoning, risk classification, and suggested alternatives
- **Separation of Duties** — agent owners cannot approve their own agent's requests

## Audit Trails

**FINRA requires:** Logging AI agent actions and decisions, implementing guardrails.

**SidClaw provides:**
- **Correlated Traces** — every evaluation produces a chronological event chain
- **Integrity Hashes** — tamper-proof hash chain on audit events
- **SIEM Export** — JSON and CSV export for compliance teams, continuous webhook delivery
- **Trace Verification** — API endpoint validates hash chain integrity
```

Create similar pages for EU AI Act (Articles 9, 12, 13, 14) and NIST AI RMF.
