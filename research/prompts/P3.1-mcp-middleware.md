# Task: P3.1 — MCP Governance Middleware

## Context

You are working on the **Agent Identity & Approval Layer** project. Read these files first:

1. `research/2026-03-20-product-development-plan.md` — read **P3.1** section carefully, especially the approval_required behavior note.

3. `packages/sdk/src/errors.ts` — SDK error classes.

Your job is to build an MCP governance server that wraps any existing MCP server and intercepts tool calls, evaluating them against the policy engine before forwarding.

## Critical Design Decision

**The MCP governance server CANNOT block waiting for human approval.** MCP `tools/call` is request-response — blocking for minutes/hours would timeout. Instead:

- On `allow`: forward to upstream, return result, record outcome
- On `approval_required`: return a structured MCP error with trace_id and approval_request_id. The agent/orchestrator handles the wait.
- On `deny`: return MCP error immediately
- Optional `approvalWaitMode: 'block'` config allows short waits (30s max) for automated approval scenarios

### 4. Tool Mapper (`tool-mapper.ts`)

```typescript
/**
 * Matches a tool name against a ToolMapping using glob-like patterns.
 * Supports: exact match, trailing wildcard ("db_*"), leading wildcard ("*_query").
 */
export function findMapping(toolName: string, mappings: ToolMapping[]): ToolMapping | undefined {
  // Exact match first
  const exact = mappings.find(m => m.toolName === toolName);
  if (exact) return exact;

  // Glob match
  for (const mapping of mappings) {
    if (mapping.toolName.includes('*')) {
      const regex = new RegExp('^' + mapping.toolName.replace(/\*/g, '.*') + '$');
      if (regex.test(toolName)) return mapping;
    }
  }

  return undefined;
}

/**
 * Derives resource_scope from tool arguments when no explicit mapping exists.
 */
export function deriveResourceScope(toolName: string, args: Record<string, unknown>): string {
  const scopeKeys = ['path', 'file', 'table', 'database', 'collection', 'bucket', 'resource', 'url', 'endpoint'];
  for (const key of scopeKeys) {
    if (typeof args[key] === 'string') return args[key] as string;
  }
  return toolName;
}
```

### 5. Tool Interceptor (`tool-interceptor.ts`)

Handles the governance evaluation for a single tool call:

```typescript
import { AgentIdentityClient } from '../client/agent-identity-client';
import type { GovernanceMCPServerConfig, ToolMapping } from './config';
import { findMapping, deriveResourceScope } from './tool-mapper';

interface InterceptResult {
  action: 'forward' | 'error';
  traceId?: string;
  error?: { code: number; message: string; data?: Record<string, unknown> };
}

export async function interceptToolCall(
  toolName: string,
  args: Record<string, unknown>,
  client: AgentIdentityClient,
  config: GovernanceMCPServerConfig,
  upstreamServerName: string
): Promise<InterceptResult> {
  const mapping = findMapping(toolName, config.toolMappings ?? []);

  // Skip governance if configured
  if (mapping?.skip_governance) {
    return { action: 'forward' };
  }

  // Build evaluation request
  const evalRequest = {
    operation: mapping?.operation ?? toolName,
    target_integration: mapping?.target_integration ?? upstreamServerName,
    resource_scope: mapping?.resource_scope ?? deriveResourceScope(toolName, args),
    data_classification: mapping?.data_classification ?? config.defaultDataClassification ?? 'internal',
    context: { mcp_tool: toolName, mcp_args: args, mcp_server: upstreamServerName },
  };

  const decision = await client.evaluate(evalRequest);

  if (decision.decision === 'allow') {
    return { action: 'forward', traceId: decision.trace_id };
  }

  if (decision.decision === 'deny') {
    return {
      action: 'error',
      traceId: decision.trace_id,
      error: {
        code: -32001,
        message: `Action denied by policy: ${decision.reason}`,
        data: {
          type: 'action_denied',
          trace_id: decision.trace_id,
          reason: decision.reason,
          policy_rule_id: decision.policy_rule_id,
        },
      },
    };
  }

  // approval_required
  if (config.approvalWaitMode === 'block') {
    // Block mode: wait for approval with short timeout
    try {
      const approval = await client.waitForApproval(decision.approval_request_id!, {
        timeout: config.approvalBlockTimeoutMs ?? 30000,
        pollInterval: 1000,
      });
      if (approval.status === 'approved') {
        return { action: 'forward', traceId: decision.trace_id };
      }
      // Denied or expired
      return {
        action: 'error',
        traceId: decision.trace_id,
        error: {
          code: -32001,
          message: `Approval ${approval.status}: ${approval.decision_note ?? 'No reason provided'}`,
          data: {
            type: `approval_${approval.status}`,
            trace_id: decision.trace_id,
            approval_request_id: decision.approval_request_id,
          },
        },
      };
    } catch {
      // Timeout — return as error
      return {
        action: 'error',
        traceId: decision.trace_id,
        error: {
          code: -32001,
          message: `Approval required but timed out waiting: ${decision.reason}`,
          data: {
            type: 'approval_required',
            trace_id: decision.trace_id,
            approval_request_id: decision.approval_request_id,
            reason: decision.reason,
          },
        },
      };
    }
  }

  // Error mode (default): return immediately with structured error
  return {
    action: 'error',
    traceId: decision.trace_id,
    error: {
      code: -32001,
      message: `Approval required: ${decision.reason}`,
      data: {
        type: 'approval_required',
        trace_id: decision.trace_id,
        approval_request_id: decision.approval_request_id,
        reason: decision.reason,
      },
    },
  };
}
```

### 6. Governance Server (`governance-server.ts`)

Build the main server class using `@modelcontextprotocol/sdk`. The server:
- Acts as an MCP server (agents connect to it)
- Connects to an upstream MCP server as a client
- Proxies `tools/list`, `resources/*`, `prompts/*` directly to upstream
- Intercepts `tools/call` through the governance evaluation

```typescript
import { Server } from '@modelcontextprotocol/sdk/server/index.js';
import { StdioServerTransport } from '@modelcontextprotocol/sdk/server/stdio.js';
import { Client } from '@modelcontextprotocol/sdk/client/index.js';
import { StdioClientTransport } from '@modelcontextprotocol/sdk/client/stdio.js';
import type { GovernanceMCPServerConfig } from './config';
import { interceptToolCall } from './tool-interceptor';

export class GovernanceMCPServer {
  private server: Server;
  private upstreamClient: Client;
  private config: GovernanceMCPServerConfig;

  constructor(config: GovernanceMCPServerConfig) {
    this.config = config;

    // Create the MCP server that agents connect to
    this.server = new Server(
      { name: 'agent-identity-governance', version: '0.1.0' },
      { capabilities: { tools: {} } }  // will be populated from upstream
    );

    // Create the MCP client that connects to the upstream server
    this.upstreamClient = new Client(
      { name: 'agent-identity-governance-client', version: '0.1.0' },
      { capabilities: {} }
    );

    this.setupHandlers();
  }

  private setupHandlers() {
    // tools/list: proxy to upstream
    this.server.setRequestHandler(/* ListToolsRequestSchema */, async () => {
      const result = await this.upstreamClient.listTools();
      return result;
    });

    // tools/call: intercept with governance
    this.server.setRequestHandler(/* CallToolRequestSchema */, async (request) => {
      const { name: toolName, arguments: args } = request.params;

      const result = await interceptToolCall(
        toolName,
        args ?? {},
        this.config.client,
        this.config,
        'upstream'  // TODO: get upstream server name from initialization
      );

      if (result.action === 'error') {
        throw new McpError(result.error!.code, result.error!.message, result.error!.data);
      }

      // Forward to upstream
      const upstreamResult = await this.upstreamClient.callTool({ name: toolName, arguments: args });

      // Record outcome
      if (result.traceId) {
        this.config.client.recordOutcome(result.traceId, {
          status: 'success',
          metadata: { result_type: typeof upstreamResult },
        }).catch(() => {});  // fire and forget
      }

      return upstreamResult;
    });

    // resources/* and prompts/*: proxy directly (no governance)
    // Set up handlers for listResources, readResource, listPrompts, getPrompt
    // that simply proxy to upstream
  }

  async start() {
    // Connect to upstream server
    if (this.config.upstream.transport === 'stdio') {
      const transport = new StdioClientTransport({
        command: this.config.upstream.command!,
        args: this.config.upstream.args,
      });
      await this.upstreamClient.connect(transport);
    }
    // TODO: SSE and streamable-http transports

    // Start the governance server (listens on stdio)
    const transport = new StdioServerTransport();
    await this.server.connect(transport);
  }

  async stop() {
    await this.server.close();
    await this.upstreamClient.close();
  }
}
```

**Note:** The exact MCP SDK API may differ from what's shown above. Read the `@modelcontextprotocol/sdk` source/docs to use the correct request handler registration patterns, schema names, and transport APIs. The key behavior is what matters — intercept tool calls, proxy everything else.
