## What To Do

### 1. Tenant-Scoped Prisma Extension (`apps/api/src/db/tenant-client.ts`)

```typescript
import { PrismaClient, Prisma } from '@prisma/client';

/**
 * Creates a tenant-scoped Prisma client that automatically filters all queries
 * by tenant_id and sets tenant_id on all creates.
 *
 * Models WITHOUT tenant_id (Tenant, BackgroundJob) are excluded from scoping.
 */
export function createTenantClient(basePrisma: PrismaClient, tenantId: string) {
  const UNSCOPED_MODELS = ['Tenant', 'BackgroundJob'];

  return basePrisma.$extends({
    query: {
      $allModels: {
        async $allOperations({ model, operation, args, query }) {
          // Skip unscoped models
          if (UNSCOPED_MODELS.includes(model!)) {
            return query(args);
          }

          // Add tenant_id filter to read/update/delete operations
          const filterOps = [
            'findMany', 'findFirst', 'findFirstOrThrow', 'findUnique',
            'findUniqueOrThrow', 'count', 'aggregate', 'groupBy',
            'updateMany', 'deleteMany', 'update', 'delete', 'upsert'
          ];
          if (filterOps.includes(operation)) {
            args.where = { ...args.where, tenant_id: tenantId };
          }

          // Set tenant_id on create operations
          if (operation === 'create') {
            args.data = { ...args.data, tenant_id: tenantId };
          }
          if (operation === 'createMany') {
            const data = args.data;
            args.data = Array.isArray(data)
              ? data.map((d: any) => ({ ...d, tenant_id: tenantId }))
              : { ...data, tenant_id: tenantId };
          }
          if (operation === 'upsert') {
            args.create = { ...args.create, tenant_id: tenantId };
          }

          return query(args);
        },
      },
    },
  });
}

export type TenantPrismaClient = ReturnType<typeof createTenantClient>;
```

### 3. Update All Services and Routes

**This is the most critical part.** Every service and route handler that uses `prisma` directly must switch to `request.tenantPrisma` (or receive the tenant-scoped client as a parameter).

**Pattern for services:**

```typescript
// Before (services use global prisma):
export class AgentService {
  constructor(private readonly prisma: PrismaClient) {}
}

// After (services receive any Prisma-compatible client):
export class AgentService {
  constructor(private readonly prisma: PrismaClient | TenantPrismaClient) {}
}
```

**Pattern for routes:**

```typescript
// Before:
import { prisma } from '../db/client';
const service = new AgentService(prisma);

// After:
const service = new AgentService(request.tenantPrisma!);
```

**Services to update:**
- `AgentService` — all methods
- `PolicyService` — all methods
- `ApprovalService` — all methods
- `WebhookService` — dispatch and delivery
- `PolicyEngine` — evaluate method
- `NotificationService` — user lookup
- Routes: evaluate, traces (list, detail, export, outcome), dashboard/overview, search

**Important:** Services that are instantiated OUTSIDE of request handlers (e.g., background jobs) should continue using the unscoped `prisma` client, since they process data across all tenants (e.g., expiring approvals for all tenants, delivering webhooks for all tenants).

### 4. Explicit Tenant ID Removal

After switching to tenant-scoped clients, remove manual `tenant_id` from all query `where` clauses — the extension adds it automatically. Also remove manual `tenant_id: tenantId` from `create` data objects.

**Example cleanup:**

```typescript
// Before:
const agents = await prisma.agent.findMany({
  where: { tenant_id: tenantId, lifecycle_state: 'active' },
});

// After:
const agents = await request.tenantPrisma!.agent.findMany({
  where: { lifecycle_state: 'active' },  // tenant_id added automatically
});
```

**Be thorough.** Search for every instance of `tenant_id` in route handlers and services to verify each is either:
- Removed (because the tenant-scoped client handles it)
- Kept intentionally (because it's in a background job using unscoped prisma)
