# Task: P0.1 — Monorepo Scaffold

## Context

You are working on the **Agent Identity & Approval Layer** project — an IAM governance platform for AI agents. Read these files before doing anything:

1. `research/2026-03-20-product-development-plan.md` — the full product development plan. Read at minimum the **Overview** section (architecture decisions, repository structure) and **Phase 0** section (all P0 tasks) to understand what you're building and why.
2. `research/2026-03-20-market-viability-assessment.md` — market research (optional, for business context).

The project currently contains a **v0 prototype** (Next.js client-side demo with seeded data). Your job is to restructure the repository into a production monorepo while preserving the v0 prototype as read-only reference material.

## What To Do

Relocate the existing v0 prototype and scaffold a Turborepo monorepo with the following structure:

```
agent-identity/
├── v0-prototype/              # ALL current files moved here
├── packages/
│   ├── sdk/                   # Empty package: @agent-identity/sdk (Apache 2.0)
│   └── shared/                # Empty package: @agent-identity/shared
├── apps/
│   ├── api/                   # Empty Fastify project
│   └── dashboard/             # Fresh Next.js 15 + TypeScript + Tailwind + shadcn/ui
├── docker-compose.yml         # PostgreSQL service
├── turbo.json                 # Build/test/lint pipelines
├── package.json               # Root workspace
├── tsconfig.base.json         # Shared strict TypeScript config
└── research/                  # Existing research docs (don't move)
```

### Step-by-step

1. **Move v0 prototype.** Move everything in the project root (app/, components/, lib/, public/, and all config files like next.config.ts, tailwind.config.ts, postcss.config.mjs, eslint.config.mjs, components.json, tsconfig.json, tsconfig.tsbuildinfo, package.json, package-lock.json, next-env.d.ts, IMPROVEMENTS.md, PRD.md, TICKETS.md, ui_specifications.md, README.md) into `v0-prototype/`. Do NOT move `research/`, `node_modules/`, or `.git/`. After moving, the v0 prototype should be independently runnable via `cd v0-prototype && npm install && npm run dev`.

2. **Create root workspace.** Create a root `package.json` with npm workspaces pointing to `packages/*` and `apps/*`. Create `turbo.json` with build, test, lint, and typecheck pipeline definitions. Create `tsconfig.base.json` with strict TypeScript settings that all packages/apps will extend.

3. **Create `packages/shared`.** Empty TypeScript package. `package.json` with name `@agent-identity/shared`. `tsconfig.json` extending base. `src/index.ts` exporting nothing yet. This package will hold shared types and Zod schemas in P0.2.

4. **Create `packages/sdk`.** Empty TypeScript package. `package.json` with name `@agent-identity/sdk`. `tsconfig.json` extending base. `src/index.ts` exporting nothing yet. License field set to `Apache-2.0`.

5. **Create `apps/api`.** Minimal Fastify project. Install: `fastify`, `@fastify/cors`, `tsx` (for dev). `src/server.ts` that starts Fastify on port 4000 with a `GET /health` endpoint returning `{ status: "ok", version: "0.1.0" }`. `tsconfig.json` extending base. Dev script: `tsx watch src/server.ts`.

6. **Create `apps/dashboard`.** Fresh Next.js 15 project with TypeScript, Tailwind CSS, and App Router. Initialize shadcn/ui with dark theme. Configure "Institutional Calm" design tokens from v0 (reference `v0-prototype/app/globals.css` and `v0-prototype/tailwind.config.ts` for colors, fonts). Load Inter (default body) and JetBrains Mono (monospace) via `next/font/google`. Set `<html class="dark">`. Create a single page at `/dashboard` rendering "Dashboard shell works" with correct dark-mode styling (near-black background, muted off-white text). Create `src/lib/api-client.ts` with a typed fetch wrapper that points to `NEXT_PUBLIC_API_URL` (default `http://localhost:4000`). Dashboard should be able to import from `@agent-identity/shared`.

7. **Create `docker-compose.yml`.** PostgreSQL 16 Alpine service with health check. Port 5432. Database name: `agent_identity`. Include volume for data persistence.

8. **Configure path aliases.** Ensure `@agent-identity/shared` is resolvable from `packages/sdk`, `apps/api`, and `apps/dashboard` via TypeScript path aliases and workspace references.

9. **Update `.gitignore`.** Add entries for: all `node_modules/`, all `dist/`, `.next/`, `.turbo/`, `*.tsbuildinfo`, `.env`, `.env.local`, postgres data volume.

## Acceptance Criteria

- [ ] `turbo build` succeeds across all packages/apps
- [ ] `turbo lint` passes (configure ESLint in root or per-package)
- [ ] Each package can `import {} from '@agent-identity/shared'` without errors
- [ ] `cd v0-prototype && npm install && npm run dev` works (prototype preserved)
- [ ] `apps/dashboard`: `npm run dev` → `http://localhost:3000/dashboard` renders with dark-mode styling, Inter font, near-black background
- [ ] `apps/api`: `npm run dev` → `GET http://localhost:4000/health` returns `{ status: "ok", version: "0.1.0" }`
- [ ] `docker compose up db` starts PostgreSQL and passes health check
- [ ] No TypeScript errors anywhere (`turbo typecheck` or `tsc --noEmit` in each package)
- [ ] `research/` directory is untouched at root level

## Constraints

- Do NOT install Prisma yet (that's P0.3)
- Do NOT create types or schemas yet (that's P0.2)
- Do NOT set up Vitest yet (that's P0.6)
- Do NOT build any UI components beyond the single "Dashboard shell works" page
- Keep dependencies minimal — only what's needed for the scaffold to build and run
- Follow the code style conventions from the plan: files in `kebab-case.ts`, strict TypeScript, no `any`
